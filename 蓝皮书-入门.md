## 一、Claude Code 是什么，能干嘛

Claude Code 是 Anthropic 官方的**命令行 AI 编程助手**（npm 包 `@anthropic-ai/claude-code`）。装好后在终端敲 `claude`，就能用大白话指挥它：

- 「把这个项目读一遍，告诉我它是干嘛的、入口在哪、有哪些坑」
- 「这段代码报错了，帮我看看哪里错了并改好」
- 「给这个网站加一个深色模式开关，记住用户选择」
- 「把这个老项目从 Vue2 整体迁到 Vue3，先列方案再动手」

它能**读你整个项目的所有文件、跨文件改、跑命令、自己改自己跑、还会写测试自检**，是目前最强的"AI 程序员"之一。和网页版聊天 AI 的区别：它直接在你电脑上动手干活，而且**特别擅长大型项目**——一次能 hold 住整个代码库，而不只是回答一段代码。

> ⚠️ **国内用的最大坑**：Claude Code 默认要登录 Anthropic 官方账号、走官方 API，国内注册难、还得挂梯子。本手册的解法是**不碰官方登录**，用国内可直连的虾盘云 / 国产 Key 驱动它（见第三节）。装的环节则要绕开 npm 默认源的超时和本机代理（见第二节方式一）。

## 二、安装：多种装法，从懒到勤

> 🥇 **最推荐**：用 [U-King](https://www.u-king.org/?from=claude-code) 一键装——它的内置终端已配好便携 Node + 国内镜像，点一下就好，下面的坑它都帮你填了。以下是手动方案。

### 方式一（最推荐手动法）：在命令行里装

在 U-King 内置终端，或系统的 PowerShell / CMD 里，**逐条**复制粘贴回车：

```bash
# 1) 清掉可能写死的代理（客户机的 clash/v2ray 常把 npm 拦死）
npm config delete proxy --location=global
npm config delete https-proxy --location=global

# 2) 用国内镜像装 Claude Code（关键：别用默认源，会超时）
npm install -g @anthropic-ai/claude-code --registry=https://registry.npmmirror.com

# 3) 验证：能打印版本号就成功了
claude --version
```

最后一条能打印出版本号就装好了 ✅，回 U-King 点「重新检测」。

> 💡 敲 `npm` 提示「不是内部或外部命令」？说明缺 **Node.js**，看方式二。

### 方式二：先装 Node.js

Claude Code 靠 Node 跑，没有它装不了：

- [Node.js 官网下载](https://nodejs.org/zh-cn/download)，或 [国内镜像直链 v22 Windows x64](https://npmmirror.com/mirrors/node/v22.20.0/node-v22.20.0-x64.msi)
- 选 **LTS 长期支持版**（Windows 选 `.msi`），双击一路下一步。
- **关掉所有命令行窗口重开**（让 PATH 生效），再回方式一执行。

### 方式三：官方源（不依赖国内镜像）

国内镜像偶尔同步滞后缺包时，走官方源（需能连 `registry.npmjs.org`，必要时挂代理）：

```bash
npm install -g @anthropic-ai/claude-code
claude --version
```

### 方式四（装不上的兜底）：先放开 PowerShell 脚本策略，再装

有些机器 `claude` 装好了一敲就报「无法加载 claude.ps1，禁止运行脚本」——这是 Windows 的 PowerShell 默认禁脚本。**免管理员放开一次**（只对当前用户、安全），再回方式一装：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

提示确认时输入 `Y` 回车即可。放开后 `claude` 命令就能正常跑了；实在不想改策略，也可以改用 CMD 窗口敲 `claude`。

### 方式五（最省心 · 零命令）⭐：让 U-King 全自动装

不想敲任何命令？打开 [U-King](https://www.u-king.org/?from=claude-code)，在 Claude Code 工具卡点「一键安装」——它会**自动补 Node + 用国内镜像装 + 放开脚本策略 + 接好虾盘云驱动**，装完直接能用。上面方式一到方式四的坑，它内部全帮你处理了，这是装不上时**最可靠的退路**。

> ⚠️ U-King 装的是 Anthropic 官方原版 `@anthropic-ai/claude-code`，**无任何改动**，只是替你配好了国内镜像和能用的模型驱动。

## 四、跑通第一个任务

打开终端（U-King 内置终端，或系统 PowerShell），`cd` 进到任意一个文件夹，敲：

```bash
claude
```

进入对话后，用**中文**直接说需求，例如：

> 在当前文件夹新建一个 `hello.py`，打印「你好，Claude Code」，然后运行它给我看结果。

Claude Code 会列出它打算做的事、请你确认、然后动手写文件并运行。第一次跑通后，你就有了一个住在终端里、能 hold 住整个项目的 AI 程序员。进阶玩法（大型项目改造、整库重构、接私活变现）见 **[📕 红皮书](./红皮书-进阶.md)**。

## 五、常见报错 TOP5

<details>
<summary><b>1. npm 报 ETIMEDOUT / 一直卡住不动？</b></summary>

最常见的坑。多半是没用国内镜像，或客户机装了代理（clash / v2ray）把 npm 拦死。先跑方式一第 1 步清代理、再带上 `--registry=https://registry.npmmirror.com` 重装；还不行 `npm cache clean --force` 清缓存重试。
</details>

<details>
<summary><b>2. 提示「无法加载 claude.ps1，因为在此系统上禁止运行脚本」？</b></summary>

PowerShell 默认禁脚本。免管理员放开一次（只对当前用户）：
```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```
提示确认时输入 `Y` 回车。或改用 CMD 敲 `claude`。
</details>

<details>
<summary><b>3. 敲 npm 提示「不是内部或外部命令」？</b></summary>

说明缺 **Node.js**。照方式二装好 Node（选 LTS `.msi`），**关掉所有命令行窗口重开**让 PATH 生效，再回方式一执行。
</details>

<details>
<summary><b>4. 装好了，U-King 还说没装？</b></summary>

命令行里 `claude --version` 能打印版本号才算真装好。确认后回 U-King 点「重新检测」；还认不出就**彻底关闭 U-King 再重开**（托盘右键退出 → 双击 U-King.exe）。
</details>

<details>
<summary><b>5. 装完后怎么用？要登录 Anthropic 吗？</b></summary>

**不用登录 Anthropic。** U-King 已把底层 API 指向国内可用的虾盘云驱动（Claude Code 走 DeepSeek-V4 Pro 满血）。终端直接敲 `claude` 就能开聊，额度由虾盘云提供（「AI 设置」可查余额、充值）。当然你也可以在「AI 设置」里换成 DeepSeek / 智谱 / Kimi 或你自己的 Key。
</details>
